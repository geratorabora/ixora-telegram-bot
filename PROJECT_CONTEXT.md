# Ixora Bot Project Context

Обновлено: 2026-05-04

## Как работаем

- Общаемся на русском, на "ты".
- Рабочая схема деплоя:
  - `main` -> тестовый бот `Ixora_test_bot` на Railway.
  - `prod` -> рабочий бот `Ixora_new_bot` на Railway.
- Новые функции сначала делаем и проверяем в тестовом боте.
- В `prod` переносим только после явной отмашки.
- Перед пушем проверяем минимум:
  - текущую ветку;
  - `git status`;
  - синтаксис изменённых Python-файлов через `python -m py_compile`;
  - что в коммит не попали лишние локальные файлы.

## Что уже сделано

### Объединение спецификаций

- В разделе "Для сотрудников" добавлен мастер `Объединить спецификации`.
- Табличные части берутся из XLSX-спецификаций.
- PDF используется для блока условий оплаты, подписей и печатей.
- Генерируются итоговые XLSX и PDF.
- Добавлены проверки итогов по количеству позиций, количеству штук и суммам.
- Добавлены кнопки вместо ручных команд там, где это ускоряет сценарий:
  - чтение файлов;
  - генерация PDF;
  - архив;
  - формирование инвойса.
- Для загрузки нескольких файлов сделано единое статус-сообщение с задержкой, чтобы не путать пользователя пачкой сообщений.

### Формирование инвойса

- После генерации объединённой спецификации добавлена кнопка `Сформировать инвойс`.
- Excel-инвойс создаётся из итоговой Excel-спецификации заменой `Specification/Спецификация` на `Invoice/Инвойс`.
- PDF-инвойс создаётся отдельно и использует статичный блок подписи/печати/реквизитов:
  - `app/assets/invoice_signature.png`.
- Для PDF подключены шрифты DejaVu, чтобы кириллица не превращалась в квадраты.

### Загрузка остатков

- Загрузка файла остатков вынесена в отдельную кнопку в разделе "Для сотрудников".
- Глобальная обработка любого загруженного файла как остатков отключена.

### Письмо об утере инвойса

- В тестовом боте добавлена новая кнопка:
  - `Письмо об утере инвойса`.
- Добавлен DOCX-шаблон:
  - `app/assets/lost_invoice_letter_template.docx`.
- Бот задаёт вопросы по полям из шаблона и формирует заполненный DOCX.
- Для блоков печати и подписи добавлена локальная SQLite-база:
  - `storage/signatures.sqlite3`.
- Если блок подписи для компании и типа документа уже есть, бот берёт его из базы.
- Если нет, бот просит PNG/JPG, сохраняет его и затем формирует письмо.

## Что осталось проверить

- В тестовом боте пройти сценарий `Письмо об утере инвойса` полностью:
  - первый запуск для новой компании, когда бот просит картинку подписи;
  - повторный запуск для той же компании, когда подпись уже берётся из базы.
- Проверить внешний вид итогового DOCX:
  - все скобочные поля заменены;
  - подпись/печать вставлена в правильное место;
  - документ открывается в Word без повреждений.
- Если внешний вид шаблона поплыл после подстановки, доработать генерацию DOCX.
- После подтверждения теста перенести изменения из `main` в `prod`.
